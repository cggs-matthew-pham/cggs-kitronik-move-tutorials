# Designing with the Move Motor

These guides are **reference material** for designing your own Move Motor project. They're different from the tutorials:

- **Tutorials** teach you *how to build* specific techniques in MakeCode
- **Guides** help you decide *what to build* and *which techniques to use*

Start here when you're planning your project, not when you're looking up a block.

## Pick your entry point

### [Choosing Your Techniques](choosing-your-techniques.md)
You've done some tutorials and you're wondering which ones you actually need for your scenario. This guide groups real-world scenarios into five families and shows which tutorials support each.

### [Communication](communication.md)
How the Move and your remote actually talk to each other. Covers radio groups, channels, and roles. Read this before designing anything where devices need to stay out of each other's way.

### [Movement](movement.md)
What the Move can actually do - drive, turn, spin, react to line sensors, react to obstacles. Read this when you're sketching behaviours and need to know what's possible.

### [Combining Techniques](combining-techniques.md)
Scenarios rarely need just one technique. This guide shows how the patterns from different tutorials fit together - state + radio + sensors + menus - so you can plan a system, not just a behaviour.

### [From Big to Small](from-big-to-small.md)
You've picked a scenario. Now what? This guide walks you through decomposing a big idea ("bot that checks the school for spills") into small buildable pieces, so your plan is actually codable.

## Remember

The tutorials are **tools**, not a script. You're not graded on how many you complete - you're graded on how well you use what you've learned to solve the problem you chose.

A project using T1-T4 done thoughtfully beats a project using T1-T6 copy-pasted. The guides here are about thoughtful.
